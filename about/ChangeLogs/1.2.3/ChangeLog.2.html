    Actually also some legal, but odd, characters get replaced. Hmm. There are problems here related to i18n. In multi-byte locales g_get_user_name() presumably, at least currently, returns a multi-byte string encoded in some Windows multi-byte code page, and the code here will think that some or most characters in it are "odd", and replace them with underscores. This will cause different user names in Japanese, for instance, that happen to have the same length, to be convered to the same string of underscores.)
